# Резервирование и восстановление

Мы все недооцениваем важность резервного копирования, до того момента, когда нужно восстановиться из него. Резервное копирование бесплатно, вы можете делать неограниченное количество резервных копий. Если у вас сложная логика бота, на которую вы потратили много времени, резервировать необходимо.

{% embed url="https://youtu.be/d9zQDkIl8bU" %}

## Как дублировать проект

**Как происходит резервирование(дублирование):**

Создается новый проект и в него переносится схема бота, быстрые ответы и ответы по тексту, без каких-либо изменений. Также резервируются настройки CRM, но не данные из нее.&#x20;

{% hint style="info" %}
Копируются настройки проекта, но не копируются клиенты, заявки и интеграции(в частности с AmoCRM и Bitrix), минилендинги. Поэтому если вы сделали шаблонного бота и собираетесь его продавать, то просто скопируйте его, покупатель подключит его к себе и не увидит ваших конфиденциальных данных.
{% endhint %}

Резервирование можно выполнить двумя способами.&#x20;

Первый: во вкладке "Проекты" достаточно нажать на кнопку "Дублировать".

<figure><img src="../../.gitbook/assets/Снимок экрана 2025-04-23 в 17.55.21.png" alt="" width="355"><figcaption></figcaption></figure>

Второй вариант:&#x20;

Создать новый проект, перейти в раздел "Бэкап":

<figure><img src="../../.gitbook/assets/Снимок экрана 2025-10-23 в 14.23.07.png" alt=""><figcaption></figcaption></figure>

Далее выберите проект, из которого необходимо восстановить данные.

<figure><img src="../../.gitbook/assets/Снимок экрана 2025-04-23 в 17.57.46.png" alt="" width="563"><figcaption></figcaption></figure>

Здесь же можно скачать схему бота (она будет скачана в файл с расширением .json) и восстановить копию из выбранного проекта.

{% hint style="warning" %}
При восстановлении вся схема проекта будет удалена и скопирована из выбранного проекта.
{% endhint %}

## Как копировать схемы бота в проекте

Для копирования схемы нужно во вкладке "Бэкап" выбрать "Копирование схемы".

{% hint style="info" %}
Ваша схема останется нетронутой, дополнительно к ней будет перенесена схема из выбранного проекта.
{% endhint %}

<figure><img src="../../.gitbook/assets/Снимок экрана 2025-04-23 в 18.00.45.png" alt=""><figcaption></figcaption></figure>

## Как восстановить схему из файла

Если есть уже бекап со схемой текущего проекта (файл с расширением .json), можно во вкладке "Восстановление из файла" добавить сохраненную в файле схему в проект. Либо если вы ранее сохраняли схему бота из другого проекта:

<figure><img src="../../.gitbook/assets/Снимок экрана 2025-04-23 в 18.03.04.png" alt="" width="563"><figcaption></figcaption></figure>

В этом случае восстановятся удаленные блоки, соединения, а также их настройки.&#x20;

<figure><img src="../../.gitbook/assets/Снимок экрана 2025-04-23 в 18.05.25.png" alt=""><figcaption></figcaption></figure>
